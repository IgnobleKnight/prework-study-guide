HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta element is added in the head element to change aspects of the webpage not seen by the end user.
  • The Img element is used to insert an image and format the image into a webpage.
    Image formatting attribues include:
    • src: Used to define the location of the image.
    • alt: Used to define a name for the image on the webpage for the end user.
    • width: Used to define a width that the image is displayed on the webpage.
    • height: Used to define a height that the image is displayed on the webpage.

  • The Text elements are used to add text to an HTML file that is visable to the end user.
    The following are examples of text elements:
    • Unformated text (Anything typed in the html file without a formatting element or css formatting will display as plain unformated text)
    • li: list items
    • ul: Unordered list
    • ol: Ordered list
    • h1-h6: Different levels of headers that are used to segment a webpage

  • The br element is used to insert a line break on the webpage.
  • The main element is used to display the main content of a webpage.
  • The footer element appears at the bottom of a webpage and usually used to display author, copyright, contact, sitemap, and navigation.
  • The link element creates the connection to an external file.
    The following are attributes within the link element:
    • rel: specifies the relationship between the current document and the linked document resource
    • href: attribute specifies the location (URL) of the external resource. This path is relitive.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: adds modifications in the current working branch to the staging area. -A: adds all changes
  • git commit -m "insert notes about changes": updates branch with the current changes in the staging area and adds notes about the changes
  • git pull origin branch-name: reveives the branch's modifications to the local environment. "origin" designates the source to pull from will be github.
  • git push origin branch-name: Push creates a new remote branch or adds new changes to an existing branch. "origin" designates the location to push to will be github.
